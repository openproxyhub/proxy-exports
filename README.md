# Open Proxy Export ‚Äì Live, Verified Proxy Lists

This repository publishes **continuously verified public proxies**, generated by an automated proxy monitoring system.  
The goal is to provide **high-quality**, **fresh**, and **easy-to-use** proxy lists with clear separation by protocol, country, and anonymity level.

Most public proxy lists online are scraped, outdated, and 80‚Äì95% dead.  
This project focuses on **quality over quantity** by only publishing proxies that are:

- Recently **UP**
- **Protocol-validated**
- In an **active lifecycle** state
- Successfully tested through real traffic
- Classified with **geo** and **anonymity** information

üõ† How the System Works (High-Level Overview)

This repository is powered by a custom backend system with several automated components.
Here is a simple overview:

1. Proxy Discovery

Proxies are added to a registry and assigned a lifecycle stage
(e.g., active_new, active_low, dead_stage_1, etc.).

2. Protocol Testing

A validator determines whether a proxy supports:

HTTP

HTTP CONNECT

HTTPS over TLS

SOCKS4

SOCKS5

Only proxies that pass protocol validation are eligible for export.

3. Uptime Monitoring

A separate uptime worker checks if the proxy is:

Reachable

Responding correctly

Stable over time

If a proxy has been DOWN too long, it is removed from exports.

4. Geo Detection

A geo worker determines:

Exit IP

Country

ASN / ISP

This is how country-based lists are generated.

5. Anonymity Checks

A capability probe tests:

Whether your real IP leaks

What headers the proxy adds

Whether the proxy reveals itself

This determines its anonymity level.

6. Exporting

A scheduled export script gathers only proxies that:

Are UP

Are protocol-validated

Are in an active lifecycle

Have valid geo/anonymity metadata

It then updates the TXT/JSON files and pushes changes to GitHub.
